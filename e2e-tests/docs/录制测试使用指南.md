# 录制测试使用指南

## 概述

测试录制功能允许您通过在浏览器中实际操作来自动生成 Playwright 测试脚本。录制的测试会保存到 `tests/recorded/` 目录，并可以通过 Dashboard 进行管理。

## 快速开始

### 1. 启动录制

1. 访问 Dashboard: http://localhost:3200
2. 点击左侧菜单 "测试录制"
3. 配置录制参数：
   - **目标 URL**: 要测试的应用地址（如 `http://10.20.3.2:9942/`）
   - **浏览器**: Chrome / Firefox / WebKit
   - **Headless 模式**: 是否无头模式运行
   - **录制超时**: 录制超时时间（秒）
4. 点击 "开始录制"

### 2. 执行操作

在打开的浏览器窗口中执行您要录制的测试操作，例如：
- 导航到页面
- 点击按钮
- 填写表单
- 选择选项
- 勾选复选框

### 3. 停止录制

完成操作后，点击 "停止录制" 按钮。

### 4. 查看和编辑

录制的操作会显示在时间轴上，生成的 Playwright 代码会显示在代码编辑器中。您可以：
- 删除不需要的操作步骤
- 直接编辑生成的代码

### 5. 保存测试

点击 "保存测试文件" 按钮，测试会保存到 `tests/recorded/` 目录。

## 管理录制的测试

### 查看已保存的测试

在 Dashboard 的"测试录制"页面下方，会显示所有已保存的测试文件列表：
- 文件名
- 文件大小
- 修改时间
- 操作按钮

### 操作选项

| 按钮 | 功能 | 说明 |
|------|------|------|
| **查看** | 加载文件内容到编辑器 | 可以查看和编辑测试代码 |
| **运行** | 执行录制的测试 | 使用 Playwright 运行测试 |
| **移动** | 移动到测试项目 | 将测试移动到正式测试目录 |
| **删除** | 删除测试文件 | 从 `tests/recorded/` 目录删除 |

---

## 将录制测试集成到正式用例

### 方法一：移动到测试项目（推荐）

1. 在测试文件列表中点击 "移动" 按钮
2. 选择目标项目：
   - **冒烟测试 (smoke)**: 基础功能测试
   - **回归测试 (regression)**: 完整回归测试套件
   - **AI 示例测试 (ai-example)**: AI 生成的测试示例

3. 点击 "移动" 确认

移动后，测试文件会被复制到对应的测试目录：
- `tests/smoke/` - 冒烟测试
- `tests/regression/` - 回归测试
- `tests/ai-example/` - AI 示例测试

### 方法二：手动复制

手动将 `tests/recorded/` 目录中的 `.spec.ts` 文件复制到其他测试目录：

```bash
# 例如：复制到 smoke 测试目录
cp tests/recorded/recorded-xxx.spec.ts tests/smoke/
```

### 方法三：直接运行

在 Dashboard 中点击 "运行" 按钮，可以直接执行录制的测试，无需移动文件。

## 命令行执行

### 运行录制的测试

```bash
npx playwright test --project=recorded
```

### 运行特定录制文件

```bash
npx playwright test tests/recorded/recorded-xxx.spec.ts
```

### 运行移动后的测试

```bash
# 运行冒烟测试中的录制测试
npx playwright test --project=smoke-chromium tests/smoke/recorded-xxx.spec.ts

# 运行回归测试中的录制测试
npx playwright test --project=regression-chromium tests/regression/recorded-xxx.spec.ts
```

## API 参考

### 获取文件列表
```bash
GET /api/recorder/files
```

### 获取文件内容
```bash
GET /api/recorder/files/:filename
```

### 删除文件
```bash
DELETE /api/recorder/files/:filename
```

### 移动文件到项目
```bash
POST /api/recorder/files/:filename/move
Body: { "targetProject": "smoke" | "regression" | "ai-example" }
```

### 运行测试
```bash
POST /api/recorder/files/:filename/run
```

## 最佳实践

1. **录制前准备**
   - 确保测试环境稳定
   - 准备好测试数据
   - 清理浏览器缓存

2. **录制时注意**
   - 操作要缓慢清晰，确保每个操作都被记录
   - 避免不必要的操作
   - 尽量使用稳定的选择器

3. **录制后处理**
   - 检查生成的代码
   - 删除冗余操作
   - 添加适当的断言
   - 添加测试描述

4. **测试维护**
   - 定期清理无效的录制测试
   - 将重要的录制测试移动到正式测试目录
   - 为测试文件添加有意义的注释

## 故障排除

### 问题：录制功能打不开
- 检查 Dashboard 服务是否运行
- 检查 WebSocket 连接是否正常
- 重启 Dashboard 服务

### 问题：保存的测试文件找不到
- 检查 `tests/recorded/` 目录
- 在 Dashboard 文件列表中查看
- 刷新页面重新加载文件列表

### 问题：录制的测试运行失败
- 检查目标 URL 是否可访问
- 检查选择器是否匹配
- 调整超时设置
- 检查是否有动态内容加载
